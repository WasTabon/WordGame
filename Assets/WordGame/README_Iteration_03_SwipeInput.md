# WordGame — Итерация 3: Свайп-инпут + сбор слова

## Что сделано

Инпут: тап по клетке → свайп по соседним → отпускание. Выбранные клетки подсвечиваются оранжевым, между ними рисуются линии, вверху экрана показывается собираемое слово. Валидации словарём ещё нет (итерация 4) — при отпускании слово просто логируется в консоль.

### Новые скрипты
- **WordBuilder.cs** — singleton. Обрабатывает выделение: старт через `HexCell.OnPointerDown`, затем в `Update` на каждом кадре делает EventSystem-raycast под `Input.mousePosition` и добавляет/убирает клетки. На release — `Debug.Log` и очистка.
- **WordPreviewUI.cs** — обёртка над `TextMeshProUGUI` вверху экрана, метод `SetWord(string)`.

### Что изменилось
- **HexCell.cs** — теперь реализует `IPointerDownHandler`. Добавлено поле `IsSelected`, метод `SetSelected(bool)`. Цвет клетки: slate (обычная) / orange (выделена) / почти прозрачный (vacant). При выделении цвет буквы становится тёмным для контраста.

### Правила инпута
- **Старт** слова возможен только на не-vacant клетке, у которой есть хотя бы один vacant-сосед.
- **Следующая клетка** должна быть соседом последней выбранной (по hex-правилам, 6 направлений), не vacant, не уже выбранной.
- **Backtrack**: палец пришёл на предпоследнюю выбранную клетку → последняя снимается.
- **Release**: если в слове ≥ 2 букв — лог `[WordBuilder] Submitted: XYZ` в консоль. Иначе тихая отмена.

## Установка

### Предусловия
Уже выполнено:
- Unity 2022.3, 2D Built-in
- TMP Essentials + DOTween Free
- Итерация 1 запущена (сцена MainMenu)
- Итерация 2 запущена (сцена Game с гридом и Hex-спрайтом)

### Шаги

1. **Распаковать архив** поверх проекта. Перезапишется `HexCell.cs`, добавятся `WordBuilder.cs`, `WordPreviewUI.cs`, `SetupSwipeInputIteration3.cs`.

2. **Дождаться компиляции.**

3. **Запустить editor-скрипт:**
   `WordGame > Setup Swipe Input (Iteration 3)`
   
   Скрипт откроет `Game.unity`, добавит `LinesContainer`, `WordPreview`, компонент `WordBuilder` на `GameController`. Выставит правильный порядок sibling-ов (Background → Lines → Grid).

4. **Открыть `MainMenu.unity`** → Play.

## Как тестировать

| Проверка | Ожидаемое |
|---|---|
| PLAY → Escape/Explore → Game-сцена | 37 клеток, центр пустой, вверху пустая область для preview |
| Мышкой (или тачем) зажать одну из 6 клеток вокруг центрального пустого и тащить | Клетка становится оранжевой, буква — тёмной, preview вверху показывает букву |
| Продолжить свайп по соседним клеткам | Каждая добавляется, между ними рисуется белая полупрозрачная линия, preview растёт (`MOTH`, `CARDS` и т.п.) |
| Свайп обратно на предпоследнюю клетку | Последняя клетка отменяется, её линия исчезает |
| Попытка начать слово на клетке без vacant-соседа (например, угловая при старте) | Ничего не происходит — slot не подсвечивается |
| Попытка продолжить на не-соседа | Игнор — клетка не добавляется |
| Release с 2+ буквами | В Console: `[WordBuilder] Submitted: ABCDE`. Всё очищается |
| Release с одной буквой | Тихая отмена, preview пустеет |
| Повторный свайп после release | Работает нормально |

Открой Console (`Window > General > Console`), чтобы видеть submit-сообщения.

## Структура Game-сцены (после iteration 3)

```
Canvas
├── Background            (sibling 0)
├── LinesContainer        (sibling 1, пустой до свайпа)
├── GridContainer         (sibling 2, 37 клеток)
├── HUD                   (верх экрана)
├── WordPreview           (под HUD, y = +600)
└── GameController        (GameController + HexGrid + WordBuilder)

EventSystem
```

Линии рисуются внутри `LinesContainer`. Он — сиблинг `GridContainer` с тем же RectTransform, стоит РАНЬШЕ грида в иерархии, поэтому линии отрисовываются под клетками. В открытых промежутках между хексагонами они видны, а на клетках — нет.

## Параметры на `GameController > WordBuilder` в инспекторе

- **Grid** — ссылка на HexGrid
- **Lines Container** — контейнер для линий
- **Preview** — WordPreviewUI
- **Line Thickness** — толщина линии (14 по умолчанию)
- **Line Color** — цвет линии (белый, 65% прозрачности по умолчанию)

## Известные ограничения итерации

- Нет валидации слов — любая последовательность букв «submit-ится» в лог.
- Нет роста vacant-зоны — центральная клетка всегда одна.
- Нет path-interpolation при очень быстром свайпе: если палец перелетает через клетку за один кадр (редко при 60fps), клетка пропускается. Добавлю в итерации 8 (juice pass), если понадобится.
- Минимальная длина слова для submit = 2. В итерации 5 поднимется до 3 вместе со словарём и number-клетками.

## Идемпотентность

Editor-скрипт можно запускать повторно — находит существующие `LinesContainer`, `WordPreview`, `WordBuilder`, переиспользует. Sibling-order выставляется каждый раз. Если что-то сломалось между запусками, можно грохнуть объект вручную — на следующем запуске он создастся заново корректно.
