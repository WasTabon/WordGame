# WordGame — Итерация 5: Number-клетки + Счёт

## Что сделано

Появилась **первая внутриигровая статистика** — счёт. На некоторых клетках теперь видны цифры, которые ограничивают минимальную длину слова, начинающегося с этой клетки.

### Number-клетки
- На поле появляются **3 number-клетки** (по умолчанию). На каждой — цифра в углу (4, 5 и т.п.).
- Цифра = минимальная длина слова, **которое можно начать с этой клетки**.
- Number-клетки выбираются только из тех клеток, где **уже размещено** слово соответствующей длины (через `placedPaths` от генератора). Это гарантия: если есть number-клетка с цифрой `5`, то на ней действительно стартует какое-то 5-буквенное слово из словаря.
- Минимальная цифра — `4` (меньше бессмысленно, т.к. большинство слов от 3 букв).

### Счёт
Формула: `score = len * (len + 5)`
| Длина слова | Очки |
|---|---|
| 3 | 24 |
| 4 | 36 |
| 5 | 50 |
| 6 | 66 |
| 7 | 84 |

**Бонус number-клетки:** если слово стартует на number-клетке → ×2 ко всему. То есть собрать 5-буквенное слово с number-клетки = 100 очков.

В preview при успешном слове показывается флэш `✓ STARE` или `✓ STARE ×2` если был бонус.

### HUD
- Слева (как раньше) — `← BACK`
- Справа сверху — название режима (теперь меньшим шрифтом)
- Справа снизу — большой оранжевый счёт. При увеличении — punch-анимация через DOTween.

### Валидация ограничения по длине
Если игрок проходит через number-клетку с цифрой `5`, но собирает слово длиной 3 — preview покажет `TOO SHORT`, поле не меняется. Логика была в `WordValidator` ещё с итерации 4 (`requiredMinLength`), теперь она реально активируется number-клетками.

Если выделение **проходит через несколько** number-клеток с разными цифрами — берётся **максимум** (т.е. слово должно быть не короче самой строгой клетки в выделении).

## Параметры

**`GameController`** в инспекторе:
- `Number Cell Count` = 3 — сколько number-клеток на поле
- `Number Cell Min Value` = 4 — минимальная цифра (меньше не ставится)

**`ScoreManager`** — никаких параметров, формула захардкожена. Если нужно поменять баланс — `AddWord` в скрипте.

**`WordValidator`** в инспекторе:
- `Min Word Length` = 2 — глобальный минимум (number-клетки могут только повысить)

---

## Важный нюанс с генератором

Number-клетка ставится только если на ней **стартует** размещённое слово (path[0] этого слова). Это значит:
- На каждой number-клетке `5` есть конкретное 5-буквенное слово из словаря, которое можно собрать с неё (но игрок может также найти другое 5+ слово).
- Если генератор уложил мало слов длиной 4+, может получиться < 3 number-клеток. В Console: `Placed 2 number cells` или `Placed 1 number cells`. Не критично.

При размере поля radius=3 и нашем словаре обычно укладывается 4-7 слов длиной 4+, так что 3 number-клеток размещаются почти всегда.

## Что НЕ сделано

- **Game-over в Explore-режиме** при отсутствии валидных слов (it.6)
- **Таймер Escape-режима** и условие победы — достижение края поля (it.7)
- **High-score сохранение** между сессиями (it.6)
- **Анимация превращения в vacant** после собранного слова (it.8 - juice pass)
