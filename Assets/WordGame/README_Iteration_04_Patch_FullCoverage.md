# WordGame — Iteration 4 Patch: Full Coverage Generator

## Что изменилось

Цель: **всё поле должно состоять только из букв спрятанных слов**. Никакого шума на нормальном поле.

### Алгоритм генерации (3 фазы)

**Phase A — Anchor.** Выбирается слово длиной 4–6 букв и укладывается **начиная от соседа центра**. Это гарантирует игроку первую точку старта.

**Phase B — Greedy.** Перебираем все слова словаря (длинные первые), пытаемся уложить каждое где-нибудь на поле. Пересечения разрешены: если на пути уже стоит та же буква — путь идёт через неё.

**Phase C — Gap fill.** Для каждой непокрытой клетки итерируем короткие слова (3–4 буквы) и пытаемся уложить **так, чтобы путь обязательно проходил через эту клетку**. Использует расширенный DFS с параметром `mustInclude`. Делается до 4 проходов.

После трёх фаз — проверка solvability (как раньше, цепочка из 6 слов).

### Retry

Цикл до **50 попыток**. Принимается только поле с:
- 100% покрытие (`uncovered == 0`)
- Цепочка решаемости ≥ 6 слов

Если за 50 попыток идеального не найдено — берётся **лучший** вариант по score (приоритет покрытию, потом solvability) и оставшиеся пустые клетки заполняются шумом + warning в Console.

На практике с 401-словным словарём идеальное поле находится за 1–5 попыток в подавляющем большинстве запусков.

### Расширенный словарь

**401 слово** (раньше 200). Все без букв `F J K Q X Z`. Распределение:
- ~230 трёхбуквенных (CAT, RUN, BIB, COB, EGG, PAW, RIM, SOY, WED, ...)
- ~130 четырёхбуквенных (RAIN, TEAR, CALL, CITY, BOAT, DROP, ...)
- ~45 пятибуквенных (STARE, TRAIN, BEACH, ALERT, BIRTH, ...)

Большой пул трёхбуквенных особенно важен для phase C — ими легко затыкать одиночные пустые клетки.

## Что увидишь в Console

**Идеальный случай (>95% запусков):**
```
[BoardGenerator] Perfect board (cover=100%, chain=6, words=14) after 1 attempts. Words: STARE, EARTH, RAIN, ...
```

**Неудачный случай (<5%):**
```
[BoardGenerator] Could not find perfect board in 50 attempts. Best: cover=35/36, chain=6, words=13. Filled gaps with noise.
```

В неудачном случае 1–2 клетки на поле могут содержать `F/J/K/Q/X/Z` — это тот самый шум-fallback. Если такое будет часто — увеличить `MAX_ATTEMPTS` в `BoardGenerator.cs`.

## Установка

1. Распаковать архив поверх проекта.
2. Дождаться компиляции.
3. **Запустить editor-скрипт: `WordGame > Setup Dictionary (Iteration 4)`** — он перезапишет `word_list.txt` с расширенным набором.
4. `MainMenu.unity` → Play.

## Параметры тюнинга в `BoardGenerator.cs`

- `MAX_ATTEMPTS = 50` — попыток
- `MAX_GREEDY_FAILURES = 60` — после стольких подряд провалов в phase B остановка (но всё равно идёт phase C)
- `MAX_GAP_FILL_PASSES = 4` — проходов phase C
- `TARGET_CHAIN = 6` — желаемая длина цепочки
- `FallbackNoise = { 'F', 'J', 'K', 'Q', 'X', 'Z' }` — буквы для emergency fallback

---

## Производительность

50 попыток * (~30мс на одну) ≈ 1.5 секунды максимум. На практике идеальное поле находится с 1-3 попыток ≈ 50–150мс. Это происходит при загрузке Game-сцены, во время fade-перехода, поэтому игрок не замечает.

Если будет ощутимая задержка на слабом устройстве — уменьшить `MAX_ATTEMPTS` до 20 (больше шансов получить fallback с шумом, но faster).
