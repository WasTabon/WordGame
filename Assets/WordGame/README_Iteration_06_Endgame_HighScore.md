# WordGame — Итерация 6: Endgame + High Score

## Что сделано

Появился полноценный **endgame loop** для Explore-режима: тупик → game over popup → restart или main menu. Добавлен лучший счёт, который сохраняется между запусками.

### Detection тупика
После каждого валидного слова проверяется: можно ли ещё собрать хоть одно слово из словаря на текущей доске. Алгоритм (`DeadlockDetector`):

1. Найти все стартовые клетки (не vacant + хотя бы один vacant-сосед)
2. Для каждой стартовой клетки взять буквы → выбрать слова словаря, начинающиеся с этой буквы
3. Для каждого слова — DFS по соседям, чтобы найти цепочку клеток с правильными буквами
4. Учитываются: number-клетки (минимальная длина), уже использованные слова

Как только хоть одно слово найдено — return true (быстрый exit). На типичной доске это занимает миллисекунды.

### Game Over popup
Большой попап в центре экрана:
- Заголовок `NO MORE WORDS`
- Огромный финальный счёт
- Бейдж `★ NEW BEST! ★` если поставлен новый рекорд (зелёный)
- Текущий лучший счёт `Best: NNN`
- Кнопки `PLAY AGAIN` (рестарт Game-сцены) и `MAIN MENU`

### High Score
- Хранится в PlayerPrefs (ключи `WG_HighScore_Explore`, `WG_HighScore_Escape`)
- Отдельно для каждого режима
- Автоматически обновляется при показе game over
- Виден в попапе выбора режима в главном меню (`Best: 0` под кнопкой)

### Блокировка инпута
После показа game over `WordBuilder` игнорирует свайпы — нельзя случайно начать новое слово, попап перехватывает все клики.

## Установка

1. Распаковать архив поверх проекта.
2. Дождаться компиляции.
3. Запустить: `WordGame > Setup Endgame + HighScore (Iteration 6)`
   - Скрипт правит **обе сцены**: добавляет `GameOverPopup` в Game.unity и `BestText` в Escape/Explore кнопки на MainMenu.unity.
4. `MainMenu.unity` → Play.

## Тестирование

| Сценарий | Ожидание |
|---|---|
| MainMenu → PLAY | В попапе под Escape и Explore кнопками — текст `Best: 0` (или предыдущий рекорд) |
| Game-сцена, собрать несколько слов | Счёт растёт, поле растёт |
| Продолжать собирать слова до момента, когда больше нет валидных | Появляется попап `NO MORE WORDS` с финальным счётом и бейджем `★ NEW BEST! ★` |
| PLAY AGAIN | Fade → новая Game-сцена с нуля, счёт 0, новое поле |
| MAIN MENU | Возврат в меню, в попапе выбора режима теперь виден сохранённый Best |
| Сыграть второй раз с меньшим счётом | Game over без бейджа `NEW BEST`, Best остаётся прежним |
| Полностью перезапустить игру (закрыть Editor и снова открыть) | Best сохранился — PlayerPrefs работает |

### Способ быстро дойти до тупика для теста
Поле имеет 36 клеток. Если поле плотно покрыто словами — обычно можно собрать 5-10 слов до тупика. Если хочешь увидеть game over быстрее, собирай длинные слова — они освобождают больше клеток за раз.

## Известные нюансы

- **DeadlockDetector используется только после собранного слова.** На стартовой доске deadlock-проверка не делается — там почти гарантированно есть валидные слова (генератор обеспечивает chain ≥ 6).
- **Detector не учитывает дубликаты пути.** Если слово `EAR` физически можно собрать через две разные `E` на доске, проверяется только одна (первая попавшаяся при DFS). На практике это не вызывает ложных «тупиков» — если хоть один путь существует, алгоритм его найдёт.
- **Используется реальный `WordValidator.IsAlreadyUsed`** — повторно использованные слова детектор не предлагает (как и игроку).
