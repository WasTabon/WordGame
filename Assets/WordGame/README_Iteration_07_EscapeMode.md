# WordGame — Итерация 7: Escape Mode

## Что сделано

Escape-режим теперь полностью функциональный:
- Запускается таймер (адаптивно, ~66 секунд при radius=3)
- Игрок должен **достичь края поля** vacant-зоной до истечения таймера
- При победе — попап `ESCAPED!` с бонусом за оставшееся время
- При истечении таймера — попап `TIME'S UP`
- При тупике (нет валидных слов) в Escape — попап `TRAPPED`

В Explore-режиме всё работает как раньше — таймера нет, тупик = `NO MORE WORDS`.

### Стартовое время
Формула: `escapeBaseSeconds + gridRadius * escapeSecondsPerRadius`
- Default: `30 + 3 * 12 = 66 секунд`
- Параметры в `GameController` инспекторе

### Бонус за время
Каждая оставшаяся секунда → +5 очков. Финальный счёт = `wordScore + timeLeft * 5`.

### Визуал таймера
- Виден только в Escape-режиме (отдельный объект `TimerHUD` под основным HUD)
- Формат `M:SS`, шрифт оранжевый
- При **≤ 10 секунд** — становится красным и пульсирует через DOTween (yoyo loop, scale 1.0 ↔ 1.15)
- При получении новой секунды — цвет/пульс сбрасывается, если время выросло обратно

### Win condition
После каждого валидного слова — `EscapeWinDetector.HasReachedEdge(grid)`. Возвращает true, если **хотя бы одна** vacant-клетка имеет `max(|q|, |r|, |q+r|) == gridRadius`. То есть достаточно «прорваться» в любую точку края, не нужно весь периметр.

### Game over flow
| Условие | Попап | Цвет |
|---|---|---|
| Escape: достиг края | WinPopup `ESCAPED!` | зелёный |
| Escape: таймер истёк | GameOverPopup `TIME'S UP` | оранжевый |
| Escape: тупик до края | GameOverPopup `TRAPPED` | оранжевый |
| Explore: тупик | GameOverPopup `NO MORE WORDS` | оранжевый |

Во всех случаях работает high-score логика (TrySetHighScore). Для Escape-режима highscore включает time bonus.

## Тестирование

| Сценарий | Ожидание |
|---|---|
| MainMenu → PLAY → **Escape** | В Game-сцене сверху (под основным HUD) виден таймер `1:06`, начинается отсчёт |
| MainMenu → PLAY → **Explore** | TimerHUD не виден, таймера нет |
| Escape: собрать слова, дойти vacant-зоной до края | Попап `ESCAPED!`, зелёный, виден `+ NN time bonus`, итоговый счёт = слова + бонус |
| Escape: ждать пока истечёт таймер | На последних 10 секундах текст красный и пульсирует. На 0:00 — попап `TIME'S UP` |
| Escape: собирать пока deadlock | Попап `TRAPPED` (если до края не дотянулся) |
| Explore: deadlock | Попап `NO MORE WORDS` (как было) |
| Win → MAIN MENU → PLAY → Escape снова | Best score обновился, поле заново |
| Restart на любом попапе | Новая партия, таймер с нуля |

### Как победить быстро (для теста)
- Радиус поля 3 → расстояние от центра до края 3 хекса
- Если первое слово начинается у края, оно сразу выигрывает (но генератор стартует первое слово у центра)
- Обычно нужно собрать 2-3 слова цепочкой к краю
- Длинные слова покрывают больше клеток за раз — выгоднее

## Параметры в инспекторе

**`GameController`:**
- `Escape Base Seconds` = 30 — базовое время
- `Escape Seconds Per Radius` = 12 — добавка за каждую единицу radius
- При radius=3: `30 + 36 = 66 сек`. Можно подкрутить под желаемую сложность.

**`WinPopup`:**
- `Time Bonus Per Second` = 5

**`TimerHUD`:**
- `Warning Threshold` = 10 — секунд до конца, когда включается красный пульс
- `Normal Color` / `Warning Color`

## Известные нюансы

- **Solver в BoardGenerator пока не учитывает Escape-цель** (достижение края). Гарантируется только chain ≥ 6 слов. На практике 6 слов почти всегда хватает чтобы дотянуться до края при radius=3, но в редких случаях может не хватить — игрок видит `TRAPPED` без своей вины. Если такое случится часто — допилю в it.10 (полировка).
- **Таймер работает в реальном времени.** Если игрок свернёт игру (Pause в Editor / Home button на телефоне), таймер всё равно тикает. В future iterations можно добавить проверку `Application.isFocused`, но пока не делал.
- **Time bonus считается на момент попапа win.** Если возникнет тиковый offset — это незначительно (±1 секунда).
