# WordGame — Iteration 4 Patch: Board Generator

## Что изменилось

Раньше поле заполнялось случайными буквами по Scrabble-распределению, и большинство «цепочек» не соответствовали словам в словаре. Теперь поле генерируется так, чтобы **в нём явно содержались слова из словаря**.

### Как работает генерация

1. Выбираются случайные слова из словаря (длинные первые — их сложнее уложить в забитом поле).
2. Первое слово укладывается в поле начиная **от соседа центральной vacant-клетки** — гарантирует игроку точку старта.
3. Каждое следующее слово укладывается через DFS по соседним клеткам, **разрешены пересечения**: если буква уже стоит на клетке и совпадает с буквой слова — путь продолжается через эту клетку.
4. Процесс жадный: пока укладывается — берём следующие. Останавливаемся, когда подряд несколько слов не вписались (но минимум 5 уложили).
5. **Оставшиеся клетки** заполняются «шумовыми» буквами: `F H J K Q X Y Z`. Эти буквы **не встречаются ни в одном слове** нашего словаря — поэтому случайные слова в шуме практически невозможны.

В результате на поле обычно 7–12 «спрятанных» слов с пересечениями плюс шум, который точно не сложится в валидное слово.

### Лог в Console

При старте Game-сцены увидишь сообщение:
```
[BoardGenerator] Placed 9 words: STARE, TRAIN, TRADE, RAIN, TIRE, EARN, NEW, OWN, ICE
```
Это полный список спрятанных слов — для отладки. Можно сравнить, что найдёшь во время игры.

## Что должно быть видно

- Поле теперь визуально неоднородное: половина клеток — «осмысленные» буквы (`S T A R E`, `T R A I N`, `R I V E R` и т.п.), другая половина — `F H J K Q X Y Z` (явно шум).
- В Console при загрузке Game-сцены: `[BoardGenerator] Placed N words: ...` — список того, что спрятано.
- Слова реально находятся: попробуй обвести путь рядом с центром — велик шанс попасть в слово из словаря с первой попытки.
- Шумовые клетки (`X`, `J`, `K` и т.п.) собрать в слово невозможно — каждая попытка вернёт `NOT A WORD`.

## Дополнительные параметры

В `BoardGenerator.cs` две константы:
- `MAX_FAILURES = 30` — после скольки подряд неуспехов укладки слов остановиться.
- `MIN_WORDS = 5` — минимум слов на поле, ниже которого не останавливаемся (даже если failures выше лимита).

Шумовые буквы — массив `NoiseLetters = { 'F', 'H', 'J', 'K', 'Q', 'X', 'Y', 'Z' }`. Если расширить словарь и в нём появятся слова с этими буквами — тут надо будет соответственно изменить набор шума.

## Лимиты алгоритма

- **При маленьком словаре (как наш — 100 слов)** возможны редкие случаи, когда первое слово не удаётся уложить даже у соседа центра (например, если все слова на `R` подряд блокируются). Тогда лог: `failed to place any word starting near center`. Это очень маловероятно при 100+ словах, но возможно — игра не сломается, просто там не будет слова у центра. На практике не воспроизводилось.
- DFS не делает hard-backtracking между разными первыми словами — берёт первое, что уложилось. Если хочешь больше слов — увеличь словарь (легко: добавь строки в `Resources/word_list.txt`).
- Алгоритм недетерминированный: каждый запуск даёт новое поле. Если хочешь повторяемости (для тестов) — можно добавить параметр seed; пока не делал.
